Hugo极速创建个人博客

前言

在此之前，我并不没有考虑过要开发个人博客，而且想当然的认为开发一个博客很麻烦，直到我遇到了ta。

 

过程

话说今天，逛Go论坛时，发现一个好玩的帖子，“Go语言最棒开源项目Top”，其中有大名鼎鼎的“Docker”，“ETCD”等等。

在其中发现hugo这个开源项目，据说该作者出身于Docker创始团队，Docker的部分静态页面也是采用的hugo。

简单来说，hugo是一个能够帮你快速生成静态页面的框架。快到什么程度呢。

几千个页面只需要几十秒。

 

出于兴趣，在github中做了个人博客。

而且未来可能要将博客转移到shanewni.github.io中。

先说说怎样利用hugo来生成自己的博客并上传到github中吧。

其实官方介绍已经说的相当详细了，中文官方介绍。

 

异常问题

除官网介绍外，碰到一个问题。

Mixed Content: The page at 'https://shanewni.github.io/' was loaded over HTTPS, but requested an insecure script 'http://shanewni.github.io/js/highlight.min.js'. This request has been blocked; the content must be served over HTTPS.

经常使用前端开发的可能比较熟悉，在https中跳转http会出现跨域问题。

从而导致个人博客的某些样式界面无法正常显示。

解决方法

解决方法如此：

<meta http-equiv="Content-Security-Policy" content="upgrade-insecure-requests">

解决原理

解决原理是：

自动识别http开头的链接并转换为https。

这样就解决了目前出现跨域问题。

结束语

如此一来，开发个人博客只需几分钟，不知道这样的速度算不算是极速呢？。
